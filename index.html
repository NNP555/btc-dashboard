Ядро моделей закрыто бесплатными источниками. Ниже — метрики, которые ещё не участвуют в баллах: часть бесплатна, но требует доработки источника; часть платная и подключится в облачной версии. Веса указаны как «зарезервированные» — на сколько метрика потянет балл, когда включится.

● Бесплатно — в работе (источник есть, нужен точный слаг/расчёт)

  • NRPL / SOPR по когортам (LTH & STH) — реализованная прибыль/убыток и трата монет отдельно по долго- и краткосрочным держателям. Всплеск трат старых монет = распределение на вершине; капитуляция молодых = дно. Источник: bitcoin-data.com — нужен точный слаг когорты; частично уже закрыто метрикой Dormancy. Резерв веса: дно 3–4%, вершина 5%.
  • Options 25-delta skew (Deribit) — перекос вменённой волатильности пут/колл. Дорогие путы (страх) = зона дна; дорогие коллы (жадность) = зона вершины. Источник: Deribit — парсинг цепочки опционов + интерполяция к 25-дельте. Резерв веса: 2–3% (деривативы).

● Платно — в облачной версии (Glassnode / CryptoQuant)

  • Entity-adjusted exchange netflows — чистые потоки на биржи, очищенные от внутренних переводов самих бирж (самый чистый сигнал давления). Приток = продажи (→ вершина), отток = аккумуляция (→ дно). Резерв веса: 6–8%.
  • Whale netflows — потоки кошельков-китов (>1000 BTC) на биржи и обратно. Ранний сигнал крупных игроков. Резерв веса: 4%.
  • URPD (UTXO Realized Price Distribution) — плотность себестоимости монет по ценовым уровням: где сосредоточен объём «застрявших» покупок → зоны поддержки/сопротивления. Контекст, вес 0.
  • Полные HODL Waves / когортное RHODL — доля предложения по возрасту монет. Вызревание молодых→старых = аккумуляция/дно; обратное = распределение/вершина. Резерв веса: 3%.
  • LTH / STH Supply & Realized Price, STH-MVRV / LTH-MVRV — предложение и себестоимость долго/краткосрочных держателей по отдельности (сейчас есть только агрегатные MVRV-Z, NUPL, STH-цена). Резерв веса: 4–6%.

Пока эти метрики выключены, итоговый балл считается по доступным (веса перенормируются), поэтому цифры остаются корректными — просто с чуть меньшим числом подтверждений. В облачной версии платные источники подключаются автоматически и начинают участвовать в баллах.

🔌 Источники данных и живое обновление

Файл работает полностью офлайн на «запечённом» снимке (его дата — в шапке; снимок ежедневно пересобирается фоновой задачей). При открытии страница автоматически пытается подтянуть свежие данные из бесплатных публичных API прямо из браузера:

  • CoinGecko (api.coingecko.com) — цена, ATH, просадка, доминация, 365‑дн. история цен → скользящие средние, RSI, Bull Market Support Band. CORS открыт, обычно работает из браузера.
  • Alternative.me (api.alternative.me/fng) — индекс страха и жадности.
  • bitcoin-data.com / BGeometrics (bitcoin-data.com) — MVRV Z‑Score, SOPR, NUPL, Puell Multiple, Reserve Risk, реализованная цена, RHODL. Бесплатный API. CORS может блокироваться при открытии файла как file:// — тогда остаётся значение из снимка (его видно по метке «снимок»).
  • Binance Futures (fapi.binance.com) — ставка финансирования. Может быть недоступна по гео/CORS.

Если живое обновление где-то не сработало (CORS/гео-блок), просто впиши актуальное число вручную в таблицу — источник каждой метрики указан в её строке. Платный Glassnode нужен только для экзотических метрик; всё ядро закрыто бесплатными источниками.

⚠️ Это аналитический инструмент, а не финансовый совет. Дно и развороты не определяются одной метрикой — смысл дэшборда в совпадении многих независимых сигналов.
